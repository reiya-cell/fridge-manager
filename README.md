# わが家の冷蔵庫 - MR-JW52TJ テストツール

家庭内で、冷蔵庫の食材・保存場所・期限を管理するPWA対応Webアプリです。HTTPS公開後はスマートフォンのカメラを直接起動でき、ホーム画面にも追加できます。

## 初心者向けの見方

- `index.html`: 画面に何を置くか
- `styles.css`: 色、余白、スマートフォン表示などの見た目
- `app.js`: 画面遷移、期限提案、保存などの動き
- ブラウザの `localStorage` に保存するため、同じブラウザでは再度開いてもデータが残ります。サーバーやアカウント共有はまだありません。

## 現在できること

1. 食材名、数量、保存状態を入力
2. 冷蔵または冷凍に合う収納場所を選択
3. 食材名から分類と期限目安を提案（ユーザーが修正可能）
4. 左メニューから中身と3日以内の期限を確認
5. 食材の検索、場所の絞り込み、削除
6. HTTPS環境でカメラを直接起動して撮影
7. 区画ごとの使用容量と残量を確認
8. レシートを端末内OCRで読み取り、複数食材を確認して一括追加
9. JAN/EANバーコードを読み取り、商品を単品入力へ反映

## ローカル確認

`index.html` をダブルクリックして基本画面を確認できます。カメラとPWAはHTTPSまたは `localhost` が必要です。

## GitHub PagesでHTTPS公開

1. GitHubで新しいリポジトリを作成します。
2. このフォルダのファイルをリポジトリの `main` ブランチへ追加します。
3. GitHubの `Settings` → `Pages` → `Build and deployment` で `GitHub Actions` を選びます。
4. `Actions` の `Deploy HTTPS site` が完了すると、PagesのHTTPS URLが表示されます。

公開後も食材データと撮影画像は端末内で扱い、現在の版ではサーバーへ送信しません。端末やブラウザを変えると在庫データは共有されません。

## レシート一括入力

「レシートから入力」で撮影すると、無料の端末内OCRが文字を読み取ります。候補と数量を選んだ後、食材ごとに1件ずつ保存状態と収納場所を決め、最後にまとめて追加します。

OCR結果のうち `FOOD_LIST.txt` に登録された食材名・別名と一致したものだけが候補になります。認識できない商品は推測せず、通常の手入力で追加します。初回OCR時は文字認識データの読み込みに通信が必要です。

## バーコード入力

「バーコードで追加」から、45または49で始まる日本向けJANコードを読み取り、Open Food Factsの日本向け無料商品データベースで商品名を検索します。海外製品のコードは対象外です。データベースに未登録の商品は商品名を手入力します。バーコード番号は商品検索のため外部データベースへ送信されますが、撮影映像は送信しません。

一度検索または手入力したJANコードと商品名はこの端末内に保存され、次回から外部検索を行いません。しょうゆ、味噌、マヨネーズなど一般的な冷蔵調味料は、バーコード、商品ラベルOCR、通常入力に対応します。

## 自動検証

同梱Node.js環境では `node tests/logic-simulations.mjs` を実行します。結果と手動確認項目は `CHANGE_HISTORY.txt` に記録しています。

## 注意

期限と占有容量は一般的な目安であり、安全性や実容量を保証しません。商品の表示、保存状態、におい・見た目を優先してください。
